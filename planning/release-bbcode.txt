[b]stolmine firmware + habitat v2.2.0[/b]

Bug-fix release. No new units; [b]spreadsheet[/b] bumped 2.1.0 -> 2.2.0, all other packages unchanged.

[b]Tomograph -- hardware viz fix[/b]

Restored the "ferrofluid ring" overview visualization on ER-301 hardware. The radial polyline had been rendering with a distended bottom lobe and spoke lines landing in wrong positions since the v2.0.0 window. Root cause was that runtime sinf/cosf calls from inside a package .so miscompute on am335x with the TI 4.9.3 toolchain; firmware-compiled graphics (ScaleQuantizer's PitchCircle, etc.) are unaffected because they link directly. Fixed by rebuilding FilterResponseGraphic around a 72-entry precomputed cos/sin lookup table. Identical render math, values never touch libm at runtime. Emulator behavior unchanged.

[b]New: Petrichor pitch macro[/b]

Sixth MacroControl on Petrichor alongside vol / pan / cutoff / Q / type. 14 presets that snap all active taps to musical pitch patterns in semitones: unison at 0, +/-12, +/-7; ascending and descending semitones; octave spreads; even/odd splits at octave and fifth intervals; major and minor chord cycles. Lives in the taps expansion row as button [b]pitch[/b]. Sets per-tap pitch non-destructively so further per-tap edits work over the macro baseline.

[b]Serialization overhaul[/b]

Every unit in the spreadsheet package now round-trips its full user-facing state across quicksave / reload. Previous behavior was that most top-level faders reverted to insert-time defaults on reload, and some toggle state and preset labels stayed out of sync with the underlying value.

[b]Step-list UX[/b]

Excel / Ballot / Larets / Etcher: reducing step count below the currently selected step now drags the cursor to the new last entry and the viewport follows, instead of scrolling past the end and leaving the list rendering as a blank region. On re-entering the ply the edit buffer reconciles to the clamped step.

sudo rm -rf /mnt/v0.7/libs/spreadsheet/
[/code]
